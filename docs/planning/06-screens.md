# Screens: IRIS 자동 매칭 앱

> 이 문서는 /screen-spec의 입력으로 사용됩니다.

## 하단 탭바
홈 / 매칭 / 보고서 / 설정

---

## 1. 온보딩 화면 (3장)

- **진입**: 앱 최초 실행
- **요소**:
  - 페이지 1: 앱 소개 (IRIS 자동 매칭이란?)
  - 페이지 2: 핵심 기능 안내 (사업자번호 → AI 매칭)
  - 페이지 3: 시작하기 버튼
- **이동**: 시작하기 → 사업자번호 입력 화면
- **인증**: 불필요

## 2. 사업자번호 입력 화면

- **진입**: 온보딩 완료
- **요소**:
  - 사업자번호 입력 필드 (10자리, 숫자만)
  - 조회 버튼
  - 로딩 인디케이터 (API 조회 중)
- **액션**: 조회 → 공공API 호출 → 프로필 설정으로 이동
- **에러**: 유효하지 않은 번호, API 실패 시 에러 메시지
- **이동**: 조회 성공 → 프로필 설정

## 3. 기업 프로필 설정 화면

- **진입**: 사업자번호 조회 성공, 설정 탭에서 프로필 편집
- **요소**:
  - 자동 채움 영역: 회사명, 대표자, 업종, 매출, 직원수, 소재지
  - 사용자 입력 영역: 연구분야 (Multi-select), 기술 키워드 (태그 입력)
  - 저장 버튼
- **데이터**: 공공API 조회 결과 + 사용자 입력
- **이동**: 저장 → 홈 화면

## 4. 홈 화면 (탭 1: 홈)

- **진입**: 앱 시작 (프로필 등록 후), 하단 탭
- **요소**:
  - 인사 메시지 ("OO기업님, 오늘의 매칭 결과입니다")
  - 적합 공고 수 카드
  - 마감 임박 공고 리스트 (상위 3개)
  - 최신 매칭 결과 요약 카드
  - 새로고침 (pull-to-refresh)
- **이동**:
  - 공고 카드 탭 → 공고 상세
  - "전체 보기" → 매칭 탭

## 5. 매칭 목록 화면 (탭 2: 매칭)

- **진입**: 하단 탭
- **요소**:
  - 검색/필터 바 (키워드, 진행 중만, 기간)
  - 정렬 옵션 (적합도순, 마감일순, 최신순)
  - 매칭 카드 목록:
    - 공고명
    - 적합도 점수 (% 게이지)
    - 마감일 (D-day 뱃지)
    - 공고기관명
  - 로딩/빈 상태 처리
- **이동**: 카드 탭 → 공고 상세

## 6. 공고 상세 화면

- **진입**: 매칭 목록 또는 홈에서 공고 카드 탭
- **요소**:
  - 상단: 공고명 + 공고기관 + 마감일
  - 적합도 점수 (원형 프로그래스 + 분석 근거)
  - AI 요약 섹션
  - 공고 전문 (접힘/펼침)
  - 첨부파일 목록 (다운로드 버튼)
  - 보고서 PDF 다운로드 버튼
  - 전문가 연결 버튼 (CTA)
- **이동**:
  - 전문가 연결 → 전문가 상담 폼
  - 뒤로 → 매칭 목록

## 7. 전문가 상담 폼 화면

- **진입**: 공고 상세에서 "전문가 연결" 버튼
- **요소**:
  - 자동채움: 회사명, 사업자번호, 관심 공고명, 적합도 점수
  - 사용자 입력: 이름, 이메일, 연락처, 문의 내용
  - 제출 버튼
  - 제출 성공 → 확인 다이얼로그 ("이메일로 확인 메일이 발송되었습니다")
- **데이터**: 프로필 + 공고 정보에서 자동 채움
- **이동**: 제출 성공 → 공고 상세로 복귀

## 8. 보고서 탭 (탭 3: 보고서)

- **진입**: 하단 탭
- **요소**:
  - 다운로드한 보고서 목록
  - 각 항목: 공고명, 적합도, 생성일, PDF 아이콘
  - 탭하면 PDF 뷰어 또는 공유
  - 빈 상태: "아직 보고서가 없습니다"
- **이동**: 항목 탭 → PDF 뷰어/공유

## 9. 설정 화면 (탭 4: 설정)

- **진입**: 하단 탭
- **요소**:
  - 프로필 편집 (→ 프로필 설정 화면)
  - 알림 설정 (자동 검색 ON/OFF, 주기)
  - 앱 버전 정보
  - 로그아웃/데이터 초기화
- **이동**: 프로필 편집 → 프로필 설정 화면
