## 2. 기술 스택

### 결정 방식
- **사용자 레벨**: L3 (경험자)
- **결정 방식**: 사용자 직접 선택

### 2.1 프론트엔드

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Flutter | 멀티플랫폼(iOS/Android) 동시 지원, 사용자 지정 |
| 상태관리 | Riverpod 또는 Provider | Flutter 공식 권장 |
| HTTP 클라이언트 | Dio | 인터셉터, 에러 핸들링 |

### 2.2 백엔드

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | FastAPI (Python) | 스크래핑/AI 생태계 최적, 자동 API 문서 |
| 스크래핑 | BeautifulSoup + httpx (또는 Selenium) | IRIS 사이트 파싱 |
| LLM | OpenAI GPT API | 기업-공고 적합도 분석 |
| 이메일 | SMTP 또는 SendGrid | 전문가 상담 확인 이메일 |
| PDF 생성 | ReportLab 또는 WeasyPrint | 매칭 보고서 PDF |

### 2.3 데이터베이스

| 항목 | 선택 | 이유 |
|------|------|------|
| DB | Notion DB | 사용자 지정, API 연동 가능 |
| 용도 | 기업 프로필, 상담 신청, 매칭 이력 저장 | |

### 2.4 외부 API

| API | 용도 |
|-----|------|
| 공공API (기업마당 등) | 사업자번호 → 기업 기본정보 조회 |
| IRIS 웹 스크래핑 | 정부지원사업 공고 수집 |
| OpenAI GPT API | 적합도 분석, 공고 요약 |
| Notion API | DB CRUD |

## 3. Decision Log

| 결정 | 대안 | 선택 이유 |
|------|------|----------|
| Flutter | React Native, Native | 멀티플랫폼, 사용자 지정 |
| FastAPI | Django, Express | 스크래핑/AI Python 생태계 |
| Notion DB | PostgreSQL, Supabase, Firebase | 사용자 지정 |
| OpenAI GPT | Claude, Gemini | 사용자 지정 |

## 4. 비기능 요구사항

| 항목 | 목표 |
|------|------|
| 스크래핑 응답시간 | < 10초 |
| LLM 분석 시간 | < 30초 |
| 앱 초기 로딩 | < 3초 |
| 오프라인 지원 | 캐싱된 매칭 결과 조회 가능 |

## 5. 보안 고려사항

- 사업자번호는 민감정보 → 전송 시 HTTPS 필수
- OpenAI API Key는 백엔드에만 저장 (앱에 노출 금지)
- Notion API Token 서버사이드 관리
